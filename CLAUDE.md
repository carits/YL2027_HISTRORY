# CLAUDE.md — OI 学生记录仓库协作规范

## 仓库用途

本仓库用于管理 OI（信息学竞赛）学生的档案、题目库、比赛记录、训练记录、代码档案等，服务于教学分析和个性化训练。

## 目录结构

```
├─ 00-使用说明/     记录规范、标签规范、代码归档规范、GPT分析使用说明
├─ 01-学生档案/     每个学生一个目录，包含总档案、知识点掌握、比赛/训练记录等
├─ 02-题目库/       题目总表、标签体系、按知识点/难度分类、重点题记录
├─ 03-比赛记录/     比赛总表、按年份存放比赛记录、复盘模板
├─ 04-训练记录/     训练总表、专题训练、周训练
├─ 05-代码档案/     按学生归档代码、共性代码问题
├─ 06-班级记录/     班级总览、学生分层、共性问题、教学计划
├─ 07-GPT分析材料/  各类分析提示词
└─ 08-模板/         所有文档模板
```

## 记录填写规范

### 文件命名

- 学生目录：`S{编号}-{姓名}`，如 `S001-张三`
- 比赛记录：`{年份}-{月份}-{比赛名称}.md`，如 `2026-05-提高组模拟赛第1场.md`
- 周训练：`{年份}年第{周数}周训练.md`，如 `2026年第19周训练.md`
- 代码文件：`{平台}_{题号}.{扩展名}`，如 `luogu_P3372.cpp`

### 日期格式

统一使用 `YYYY-MM-DD`，如 `2026-05-10`

### 内容格式

- 每个 md 文件必须有标题（一级标题）
- 表格使用 markdown 标准格式
- 标签用逗号分隔，如 `线段树, 懒标记, 区间加`

## 三维难度评分体系

每道题从三个维度评分（1-5 分）：

| 维度 | 含义 | 核心问题 |
|---|---|---|
| 知识点难度 | 需要学过什么 | 不会这个专题就做不了 |
| 思维难度 | 学过后能不能想到 | 告诉他用什么算法后会不会做 |
| 实现难度 | 想到后能不能写对 | 思路清楚但容易 WA/TLE/RE |

评分等级：1-入门 / 2-基础 / 3-提高 / 4-进阶 / 5-高阶

详细标准见 `02-题目库/标签体系.md`

## 综合等级

L1 入门 → L2 普及- → L3 普及+ → L4 提高- → L5 提高 → L6 提高+ → L7 省选- → L8 省选 → L9 省选+ → L10 NOI

## 标签体系

- **主标签**：每题只选一个（基础语法/模拟/枚举/贪心/二分/递归与搜索/动态规划/数据结构/图论/树/字符串/数学/组合计数/计算几何/博弈论/构造/交互/随机化）
- **子标签**：可多个，如 `线段树, 懒标记`
- **能力标签**：读题理解/边界处理/暴力设计/复杂度分析/模型转化/状态设计/性质观察/贪心证明/单调性判断/数据结构维护/图论建模/分类讨论/代码调试/模板熟练度/比赛策略
- **常见错误标签**：思路类/实现类/比赛类/学习类（详见标签体系文档）
- **教学标签**：模板题/入门题/巩固题/变形题/综合题/思维题/易错题/压轴题/比赛题/讲评题/二刷题/拓展题等

## Git 操作规范

- 提交信息用中文，简洁描述改动内容
- 每次提交聚焦一个主题（如"添加S003学生档案"或"更新第20周训练记录"）
- 不要提交敏感信息（密码、个人隐私等）
