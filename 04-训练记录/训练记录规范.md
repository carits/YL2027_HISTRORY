# 训练记录规范

## 一、目录结构

```
04-训练记录/
├─ 训练总表.md（索引，含训练编号）
├─ 训练记录规范.md
└─ 2026/
    ├─ T2026-001-前缀和差分/
    │   ├─ 00-训练总览.md
    │   ├─ 01-题目列表.md
    │   ├─ 02-学生完成情况.md
    │   ├─ 03-代码与错因索引.md
    │   ├─ 04-训练复盘.md
    │   └─ 05-后续安排.md
    └─ T2026-002-二分与贪心/
        └─ （同上结构）
```

## 二、训练编号规则

- 格式：`T{年份}-{序号}`
- 示例：T2026-001、T2026-002
- 目录命名：`T{年份}-{序号}-{训练主题简称}`
- 简称规则：中文，无空格，如"前缀和差分"、"二分与贪心"

## 三、文件命名规范

训练目录内的文件统一使用数字前缀命名：

| 文件名 | 内容 |
|---|---|
| 00-训练总览.md | 训练基本信息、目标、整体结果 |
| 01-题目列表.md | 题目详情、题目库编号关联 |
| 02-学生完成情况.md | 每个学生的完成状态、错因、代码链接 |
| 03-代码与错因索引.md | 代码存放路径、错因统计 |
| 04-训练复盘.md | 整体复盘、各题分析、分层判断 |
| 05-后续安排.md | 二刷、补弱、关联比赛、档案更新 |

## 四、训练类型分类

| 类型 | 说明 |
|---|---|
| 专题训练 | 针对特定知识点的系统性训练 |
| 周训练 | 日常周训练 |
| 补弱训练 | 针对薄弱点的补充训练 |
| 赛前训练 | 赛前针对性训练 |

## 五、完成状态标准化

| 状态 | 含义 |
|------|------|
| 未做 | 未尝试 |
| 尝试未过 | 有提交但未AC |
| 已完成 | 已AC |

独立程度在学生完成情况表中单独记录：

| 独立程度 | 说明 |
|------|------|
| 独立 | 无提示独立完成 |
| 提示后 | 经提示后完成 |
| 看题解 | 看题解后完成 |
| 讲解后 | 课堂讲解后完成 |
| 订正 | 订正后AC |

## 六、题目库关联

训练题目必须关联题目库内部编号：

| 序号 | 题目库编号 | 示例 |
|---:|---|---|
| 1 | P0090 | 第1题对应题目库 P0090 |
| 2 | P0102 | 第2题对应题目库 P0102 |

关联后可统计：
- 某专题训练的整体完成率
- 某学生在某类题目上的训练表现
- 某题是否已在题目库中、是否需要二刷

## 七、代码档案关联

训练代码存放路径：

```
05-代码档案/按学生/{学号}-{姓名}/训练代码/T{编号}/{题目库编号}/
```

代码命名规范：
- 普通版本：`v{版本号}-{状态}.cpp`
- 订正版本：`v{版本号}-AC-corrected.cpp`（可选标注）

## 八、与比赛记录的关联

训练记录可关联比赛记录：

- 前测比赛：训练前的基础摸底
- 后测比赛：训练效果检验

在训练总览的"关联比赛"字段填写比赛编号。

## 九、更新日志

训练总表末尾维护更新日志：

| 日期 | 变更类型 | 变更内容 | 变更人 |
|---|---|---|---|
| YYYY-MM-DD | 新增/修改 | （具体描述） | （填写） |

## 十、模板位置

训练相关模板存放在 `08-模板/` 目录：

- 训练代码索引模板.md
