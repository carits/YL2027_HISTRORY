# 数据隐私与脱敏规范

## 一、不得入库的信息

以下信息不建议写入 Git 仓库：

1. 家长联系电话。
2. 家长身份证号。
3. 家庭住址。
4. 过度详细的家庭隐私。
5. 未经确认的传闻。
6. 不必要的负面评价。

## 二、可以入库的信息

在私有仓库中，以下信息可以用于教学管理：

1. 学生姓名。
2. 性别。
3. 片区。
4. 当前学校。
5. 学习机构。
6. 推荐方。
7. 学习进度。
8. 比赛表现。
9. 约谈结论。
10. 培养状态。
11. 风险标签。
12. 后续动作。

## 三、电话脱敏规则

所有电话统一写作：

```text
已脱敏
```

## 四、负面信息记录规则

负面信息只记录事实和处理动作，不做人格化评价。

建议结构：

```text
风险类型：
事实记录：
处理方式：
后续观察：
```

## 五、公开仓库注意事项

如果仓库为公开仓库，建议进一步脱敏：

1. 学生姓名改为 S001、S002。
2. 学校只保留区域，不保留具体小学。
3. 备注中删除家庭细节。
4. 约谈记录只保留类型，不保留具体承诺。
