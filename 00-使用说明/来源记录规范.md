# 来源记录规范

## 一、记录目的

来源记录用于追踪学生入班前后的背景信息，服务于分班、教学起点判断、家长沟通、续班评估和后续招生分析。

记录重点不是评价其他机构或学生家庭，而是客观回答：

- 学生从哪里来？
- 原来学过什么？
- 入班时真实水平如何？
- 家长期望是什么？
- 学生适合什么班型和训练节奏？
- 后续需要重点跟进什么？

## 二、来源渠道固定标签

填写 `来源渠道` 时建议只使用以下标签，避免同义词混乱。

| 标签 | 含义 |
|---|---|
| 家长转介绍 | 由家长推荐报名 |
| 学生转介绍 | 由学生同学、朋友推荐 |
| 学校推荐 | 由学校、老师、社团推荐 |
| 原机构转入 | 曾在其他机构学习后转入 |
| 公开课试听 | 通过公开课、体验课进入 |
| 线上咨询 | 通过微信、公众号、朋友圈、小红书、抖音等咨询 |
| 比赛活动 | 通过比赛、讲座、营队、测评活动进入 |
| 老学员续读 | 原有学生继续报名新阶段课程 |
| 其他 | 无法归入上述类别 |

## 三、原机构记录原则

可以记录原机构名称、学习内容、学习时长和转入原因，但不要写攻击性评价。

不建议写：

> 原机构教得很差。

建议写：

> 原课程以基础语法和简单题训练为主，系统算法专题训练较少。学生入班时语法基础尚可，但提高组专题掌握不足。

## 四、入班前能力判断

入班前至少记录以下维度：

| 维度 | 建议等级 |
|---|---|
| C++语法基础 | 未接触 / 薄弱 / 基本掌握 / 较熟练 |
| 基础算法 | 未接触 / 薄弱 / 基本掌握 / 较熟练 |
| 代码实现 | 慢 / 不稳定 / 基本稳定 / 较强 |
| 读题理解 | 薄弱 / 一般 / 较好 / 突出 |
| 数学与逻辑 | 薄弱 / 一般 / 较好 / 突出 |
| 自主训练 | 无 / 较少 / 稳定 / 主动 |
| 比赛经验 | 无 / 少量 / 有一定经验 / 经验较丰富 |

每个判断最好附带证据，例如测评题表现、试听课观察、原比赛成绩、家长描述等。

## 五、试听与转化记录

试听记录建议包含：

| 日期 | 环节 | 观察结果 | 后续动作 |
|---|---|---|---|
| YYYY-MM-DD | 试听 / 测评 / 沟通 | 学生表现与家长诉求 | 是否转化、是否跟进、建议班型 |

## 六、来源质量判断

来源质量不是给学生贴标签，而是辅助教学和运营判断。

| 维度 | 可选值 |
|---|---|
| 学生基础 | 弱 / 中 / 较好 / 强 |
| 学习主动性 | 弱 / 中 / 较好 / 强 |
| 家长配合度 | 弱 / 中 / 较好 / 强 |
| 目标清晰度 | 不清晰 / 一般 / 清晰 / 很清晰 |
| 分班适配度 | 不适合 / 需观察 / 基本适合 / 很适合 |
| 续班风险 | 高 / 中 / 低 |

## 七、隐私注意事项

- 不记录身份证号、家庭住址等强隐私信息。
- 家庭情况只记录与学习支持相关的信息。
- 对学生性格、家庭、原机构的描述应客观、克制、可用于教学。
- 公开或发给 GPT 分析前建议脱敏。
