# 学生信息字段规范

## 一、基础信息字段

| 字段 | 说明 | 是否建议入库 |
|---|---|---|
| 姓名 | 学生真实姓名 | 私有仓库可入库 |
| 性别 | 男 / 女 | 可入库 |
| 片区 | 学生所在区域 | 可入库 |
| 学校 | 当前小学 | 私有仓库可入库 |
| 家长联系方式 | 电话或微信 | 不入库，统一脱敏 |
| 学习机构 | 当前或历史学习机构 | 可入库 |
| 推荐方 | 学生来源渠道 | 可入库 |
| 派位情况 | 小升初派位、倾向、去向 | 私有仓库可入库 |
| 26寒假 / 26春 | 是否参加对应阶段学习 | 可入库 |

## 二、培养状态字段

`条件` 字段用于记录学生当前跟进状态、承诺状态、风险状态或外部去向。

建议统一拆为：

| 字段 | 说明 |
|---|---|
| 当前状态 | 普通跟进、重点观察、重点培养、承诺、保校承诺、外部去向、风险关注 |
| 跟进优先级 | S / A / B / C / R |
| 是否承诺 | 是 / 否 |
| 承诺类型 | 保校、保系、重点培养、单独培养、升学协调 |
| 风险标签 | 游戏管理风险、AI代写风险、升学外流风险、沟通未响应等 |
| 后续动作 | 面谈、测评、观察、进入训练、暂缓跟进 |

## 三、能力评分字段

当前表格包含：

| 字段 | 满分 | 说明 |
|---|---:|---|
| 天赋估计 | 12 | 理解速度、抽象能力、迁移能力、自学能力 |
| 习惯 | 12 | 稳定投入、代码习惯、复盘习惯、自我管理 |

建议判断：

| 分数 | 含义 |
|---:|---|
| 10-12 | 高潜学生，适合重点培养 |
| 8-9 | 条件较好，适合重点观察或培养 |
| 6-7 | 普通基础，需要训练稳定性验证 |
| 1-5 | 不建议直接承诺重点培养 |

## 四、学习进度字段

建议把原始进度归入阶段：

| 阶段 | 说明 |
|---|---|
| S2 语法完成 | C++ 基础语法基本学完 |
| S3 普及入门 | 刚开始学习普及内容 |
| S5 普及基本完成 | 普及组常见知识基本完成 |
| S6 普及完成但缺训练 | 知识听过，但刷题和实战不足 |
| S7 普及完成 | 普及组体系较完整 |
| S9 提高进行中 | 已接触 ST 表、LCA、最短路、并查集、MST 等 |

## 五、备注字段

备注字段应保留原始事实，但建议拆成：

1. 学生性格特点。
2. 学习行为观察。
3. 家长画像。
4. 升学关注点。
5. 风险问题。
6. 教师判断。
7. 后续跟进建议。

## 六、推荐理由字段

推荐理由建议拆为：

1. 信息学基础。
2. 数学能力。
3. 竞赛经历。
4. 自学能力。
5. 学习品质。
6. 家庭支持。
7. 特殊优势。
