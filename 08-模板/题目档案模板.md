## 二、标签信息

| 字段 | 内容 |
|---|---|
| 主标签 | （只选一个一级方向，例如 枚举 / 动态规划 / 数据结构） |
| 核心知识点 | （决定知识点难度 K，例如 差分 / 线段树 / 最短路） |
| 辅助思想 | （不参与知识点难度加分，例如 区间贡献 / 乘积上界 / 反向处理） |
| 子标签 | （可多个，逗号分隔） |
| 细分标签 | （可多个，逗号分隔） |
| 能力标签 | （从能力标签列表中选择） |
| 常见错误标签 | （从常见错误标签列表中选择） |
| 教学标签 | （从教学标签列表中选择） |

填写要求：

```text
核心知识点用于决定知识点难度 K。
辅助思想只用于解释思维难点，不参与知识点难度加分。
```

示例：

```text
主标签：基础算法
核心知识点：差分
辅助思想：区间贡献, 配对约束, 修改次数统计
子标签：区间修改, 贡献统计
能力标签：模型转化, 边界处理, 复杂度分析
常见错误标签：下标混乱, 边界遗漏, 复杂度估计错误
教学标签：思维题, 讲评题, 二刷题
```

## 三、难度评估

| 维度 | 分数 | 说明 |
|---|---:|---|
| 知识点难度 K | 1-5 | （只看核心知识点的前置门槛，不看题目难不难） |
| 思维难度 T | 1-7 | （看学过知识点后，从题面想到做法的难度） |
| 实现难度 I | 1-5 | （看想到做法后，代码写稳的难度） |
| 三维难度 | K / T / I | （例如 2 / 5 / 3） |
| 综合等级 | L1-L10 | （用于教学阶段定位，不直接等于最高分） |

### 难度说明填写要求

难度说明不要写套话，要写清楚本题为什么是这个分数。

推荐写法：

```text
知识点难度 K：核心知识点是差分，属于基础算法套路，不需要高级数据结构。
思维难度 T：需要把每一对数的修改次数转化为区间贡献，这是本题主要难点。
实现难度 I：需要处理贡献区间边界、数组下标和多组数据清空，容易出现 WA。
```

### 常见校准规则

```text
辅助思想不参与知识点难度加分。
题目难在想法时，提高思维难度 T。
题目难在边界和代码时，提高实现难度 I。
不要因为题目整体难，就直接提高知识点难度 K。
```

## 八、三维难度校准检查

每次新增或修改题目难度时，检查以下问题：

```text
1. 知识点分是不是只由核心知识点决定？
2. 有没有把辅助思想误算进知识点？
3. 有没有把边界细节误算进知识点？
4. 如果 K ≥ 3，是否真的需要完整专题课？
5. 如果 T ≥ 5，是否真的存在明显模型转化？
6. 如果 T ≥ 6，是否真的需要多步性质或隐蔽建模？
7. 如果 I ≥ 4，代码是否真的有多模块复杂性？
8. 综合等级是否符合教学阶段？
9. 难度说明是否具体到本题，而不是套话？
10. 同类型题是否和已有题目分数一致？
```
