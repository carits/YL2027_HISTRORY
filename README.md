# OI 学生记录管理系统

## 简介

本仓库用于信息学竞赛（OI）教学管理，涵盖学生档案、题目库、比赛记录、训练记录、代码档案等内容，服务于教学分析、学生画像构建和个性化训练推荐。

## 目录结构

```
├─ 00-使用说明/     记录规范与使用指南
├─ 01-学生档案/     每个学生一个目录（S{编号}-{姓名}），包含10个文件
├─ 02-题目库/       题目总表（索引）、题目档案（题面/题解/AC代码/教学分析）、按知识点/难度分类、题单
├─ 03-比赛记录/     按年份记录比赛详情与复盘
├─ 04-训练记录/     专题训练与周训练记录
├─ 05-代码档案/     学生代码归档与共性代码问题
├─ 06-班级记录/     班级整体教学记录与计划
├─ 07-GPT分析材料/  AI 辅助分析提示词
└─ 08-模板/         所有文档模板
```

## 核心设计

### 三维难度评分

每道题从三个维度评分（1-5 分）：

- **知识点难度**：这题需要学过什么？
- **思维难度**：学过之后能不能想到？
- **实现难度**：想到之后能不能写对？

### 综合等级

L1 入门 → L2 普及- → L3 普及+ → L4 提高- → L5 提高 → L6 提高+ → L7 省选- → L8 省选 → L9 省选+ → L10 NOI

## C27 小升初学生池

本仓库包含 C27 小升初学生池相关记录，用于沉淀小升初阶段信息学竞赛学生的基础信息、学习进度、竞赛表现、性格习惯、家庭沟通、升学倾向、培养状态与后续跟进记录。

记录原则：

1. 学生真实姓名、学校、机构、学习进度、推荐理由、备注、约谈结论可以进入私有仓库沉淀。
2. 家长电话不进入 Git 仓库，统一写作"已脱敏"。
3. 负面行为记录必须客观描述，避免情绪化评价。
4. 涉及承诺、保校、保系、外部学校去向等内容时，应记录事实、时间、沟通对象和后续动作。
5. 每名学生一个独立档案，便于后续进行训练安排、比赛复盘和 GPT 辅助分析。

## 快速开始

1. 查看 `00-使用说明/` 了解记录规范
2. 查看 `08-模板/` 获取各类型文档模板
3. 新增学生时参考 `08-模板/学生总档案模板.md`，按10文件结构创建
4. 新增题目时参考 `02-题目库/题目记录规范.md` 和 `08-模板/题目档案模板.md`
5. 题目总表只做索引，详细内容存放在 `02-题目库/题目档案/` 对应目录
