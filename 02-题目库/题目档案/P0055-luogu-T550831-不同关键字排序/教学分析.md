# P0055 不同关键字排序 教学分析

## 一、教学定位

| 项目 | 内容 |
|---|---|
| 主标签 | 排序 |
| 子标签 | 按列排序, 学号输出 |
| 三维难度 | 2/2/2 |
| 综合等级 | L2 |
| 适合用途 | 专题训练, 课堂讲评, 作业巩固 |

## 二、训练价值

这道题适合训练学生把题面中的过程或条件转化为可执行算法。讲评时应重点强调：为什么不能只凭样例写代码，以及如何根据数据范围选择复杂度。

## 三、讲解重点

1. 先让学生复述题意，确认输入和输出目标。
2. 找出本题的核心状态、核心公式或核心比较规则。
3. 分析朴素做法的复杂度，再引出正解。
4. 用样例手推关键过程。
5. 最后强调易错边界和代码实现细节。

## 四、适合追问的问题

- 本题的暴力做法是什么，为什么可能不够？
- 正解中维护了哪些变量？这些变量分别表示什么？
- 哪些输入会卡掉不严谨的实现？
- 代码中最容易写错的是哪一行？
