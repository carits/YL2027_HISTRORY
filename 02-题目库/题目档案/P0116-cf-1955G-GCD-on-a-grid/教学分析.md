# P0116 GCD on a grid 教学分析

## 一、教学定位

| 项目 | 内容 |
|---|---|
| 主标签 | 动态规划 |
| 核心知识点 | 动态规划 |
| 辅助思想 | 常规转化 |
| 子标签 | 动态规划, 常规转化, 路径DP, 最大公约数, 枚举因子 |
| 三维难度 | 3 / 3 / 3 |
| 综合等级 | L4 |
| 推荐阶段 | 提高- |
| 适合用途 | 易错题, 讲评题, 二刷题 |

---

## 二、难度拆解

### 1. 知识点难度 K = 3

核心知识点是 动态规划，需要系统学习标准提高组专题。

本题的知识点评分只由核心知识点决定，辅助思想不额外抬高 K 值。

### 2. 思维难度 T = 3

需要理解 常规转化，属于常规性质或坐标/状态转化。

### 3. 实现难度 I = 3

实现存在一定 WA 风险，需要稳定处理 常规转化 中的下标、边界或数据类型细节。

---

## 三、训练价值

本题适合用于训练学生对 **动态规划** 的掌握，并结合 **常规转化** 进行思路整理。讲评时应避免只给代码，应重点解释为什么该题的难点主要落在对应维度上。

---

## 四、讲解重点

1. 先让学生复述题意，确认输入、输出和限制。
2. 明确本题的核心知识点是 **动态规划**。
3. 单独拆出辅助思想：**常规转化**。
4. 对照新版三维难度解释：K 看前置知识，T 看能否想到，I 看代码是否容易写错。
5. 结合 AC 代码指出最容易出错的位置。

---

## 五、适合追问的问题

- 本题真正需要提前学过的知识点是什么？
- 哪些内容只是辅助思想，不能算进知识点难度？
- 如果学生会 动态规划，还需要想到什么才能做出本题？
- AC 代码中最容易写错的是哪一类细节？
- 本题适合作为模板题、变形题还是讲评题？
