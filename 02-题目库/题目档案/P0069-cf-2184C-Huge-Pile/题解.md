# P0069 Huge Pile 题解

## 一、题意简化

本题可以简化为：

```text
初始有一堆 n 个苹果。一次操作可以把一堆 x 个苹果分成 floor(x/2) 和 ceil(x/2) 两堆。问能否得到一堆恰好 k 个苹果，若可以，输出最少分裂次数，否则输出 -1。
```

## 二、算法设计

经过 d 次沿着某条分裂路径走下去，可能得到的堆大小只会是 floor(n/2^d) 或 ceil(n/2^d)。枚举 d，第一次命中 k 即为答案。

## 三、实现步骤

1. 读入数据。
2. 根据题意维护必要状态。
3. 按 `数学` 的核心思路计算答案。
4. 严格按题目要求输出。

## 四、复杂度分析

每组 O(log n)。

## 五、常见错误

- 没有单独处理边界数据。
- 多组数据时状态没有清空。
- 输出格式多空格、少换行或大小写错误。
- 使用了不满足数据范围的复杂度。
