# P0107 Rudolph and Christmas Tree 教学分析

## 一、教学定位

| 项目 | 内容 |
|---|---|
| 主标签 | 数学几何 |
| 核心知识点 | 数学几何 |
| 辅助思想 | 几何推导 |
| 子标签 | 数学几何, 几何推导, 几何面积, 相似三角形, 重叠扣除 |
| 三维难度 | 2 / 5 / 3 |
| 综合等级 | L4 |
| 推荐阶段 | 提高- |
| 适合用途 | 思维题, 易错题, 讲评题, 二刷题 |

---

## 二、难度拆解

### 1. 知识点难度 K = 2

核心知识点是 数学几何，属于基础算法套路；辅助思想不参与知识点加分。

本题的知识点评分只由核心知识点决定，辅助思想不额外抬高 K 值。

### 2. 思维难度 T = 5

需要把题意明显转化为 几何推导 对应的模型、贡献方式或判定方式。

### 3. 实现难度 I = 3

实现存在一定 WA 风险，需要稳定处理 几何推导 中的下标、边界或数据类型细节。

---

## 三、训练价值

本题适合用于训练学生对 **数学几何** 的掌握，并结合 **几何推导** 进行思路整理。讲评时应避免只给代码，应重点解释为什么该题的难点主要落在对应维度上。

---

## 四、讲解重点

1. 先让学生复述题意，确认输入、输出和限制。
2. 明确本题的核心知识点是 **数学几何**。
3. 单独拆出辅助思想：**几何推导**。
4. 对照新版三维难度解释：K 看前置知识，T 看能否想到，I 看代码是否容易写错。
5. 结合 AC 代码指出最容易出错的位置。

---

## 五、适合追问的问题

- 本题真正需要提前学过的知识点是什么？
- 哪些内容只是辅助思想，不能算进知识点难度？
- 如果学生会 数学几何，还需要想到什么才能做出本题？
- AC 代码中最容易写错的是哪一类细节？
- 本题适合作为模板题、变形题还是讲评题？
