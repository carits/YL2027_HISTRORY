# P0216 鸡巧 教学分析

## 一、教学定位

| 项目 | 内容 |
|---|---|
| 主标签 | 基础语法 |
| 核心知识点 | 数组查表 |
| 辅助思想 | 数据分档阅读, 二维数组预处理, 直接查表 |
| 三维难度 | 1 / 1 / 1 |
| 综合等级 | L1 |
| 推荐阶段 | 入门 |
| 适合用途 | 入门训练 / 比赛策略讲解 / 读题能力训练 |

---

## 二、题解与 AC 代码对应关系

- `kV` 是提前写好的二维分数表。
- 输入 `n, m` 后直接输出 `kV[n][m]`。
- 代码没有循环和复杂判断，正确性完全依赖分数表是否抄对。

---

## 三、难度拆解

### 1. 知识点难度 K = 1

核心操作是把题目编号和分档编号映射到二维数组中的固定分数，只需要基础数组和输入输出。

### 2. 思维难度 T = 1

题解要求仔细阅读数据分档，并把每个分档分数整理成表，思路直接，没有额外转化。

### 3. 实现难度 I = 1

AC 代码只需初始化二维常量表并按下标输出，主要风险是题号、分档编号下标写错。

---

## 四、讲解顺序建议

1. 先让学生复述题意，明确输入、输出和核心限制。
2. 对照题解拆出朴素想法，并说明为什么需要正解中的关键观察。
3. 围绕核心知识点：**数组查表** 展开算法设计。
4. 单独指出辅助思想：**数据分档阅读, 二维数组预处理, 直接查表**，避免把辅助思想误算成知识点难度。
5. 最后逐行对照 AC 代码，强调容易写错的变量、边界和维护顺序。

## 五、重点追问

- 本题真正需要提前掌握的核心知识点是什么？
- 题解里最关键的转化或观察是哪一步？
- AC 代码中哪些变量对应题解中的核心量？
- 如果直接暴力，会卡在哪个复杂度上？
- 哪些错误会导致只过样例但过不了完整数据？

---

## 六、常见错误讲评

- 没有仔细阅读每道题的数据说明，导致分档分数抄错。
- 把题目编号和题目顺序混淆，例如把【鸡巧】也当成可查询题目。
- 二维数组下标没有预留 0 位，导致 n 或 m 对应错误。
- 用大量 if 分支硬写，容易漏掉某个分档。
