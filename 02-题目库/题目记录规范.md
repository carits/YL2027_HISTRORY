# 题目记录规范

## 一、题目库设计目标

题目库不是只存题号，而是服务于以下教学场景：

- **备课**：快速找到适合当前教学阶段的题目
- **训练设计**：按知识点、难度组合训练题单
- **比赛复盘**：关联比赛中的题目，沉淀错因和教学分析
- **学生错因分析**：通过标签和三维难度定位学生薄弱环节
- **GPT 辅助生成训练计划**：提供结构化题目数据供 AI 分析
- **长期沉淀教学素材**：题面、题解、AC代码、错误代码、教学分析一体化管理

---

## 二、题目分层维护标准

### A类核心题：完整整理

适合作为课堂讲解、模板教学、重点复盘的题目。

建议包含：

```text
题目档案.md
题面.md
题解.md
AC代码.cpp
错误代码.md
教学分析.md
```

A 类题必须填写完整难度信息：

```text
核心知识点
辅助思想
知识点难度 K
思维难度 T
实现难度 I
三维难度 K / T / I
综合等级
难度说明
```

---

### B类普通训练题：中等整理

适合日常训练、课后作业的题目。

建议包含：

```text
题目档案.md
题解.md
AC代码.cpp
```

B 类题至少需要填写：

```text
主标签
核心知识点
辅助思想
三维难度 K / T / I
综合等级
整理状态
```

---

### C类索引题：只做索引

只在题目总表和相关题单中登记基本信息，不创建题目档案目录。

C 类题至少需要在题目总表中登记：

```text
内部编号
平台
题号
题目名
原题链接
VJudge写法
核心知识点
三维难度
综合等级
整理状态
```

## 四、题目文件夹命名规范

格式：

```text
{内部编号}-{平台缩写}-{原题号}-{题目名}
```

示例：

```text
P0001-luogu-P3372-线段树1
P0002-cf-242E-XOR-on-Segment
P0003-atcoder-abc123-f-题名
```

要求：

- 不使用空格
- 尽量不使用特殊符号
- 中文题名可以保留
- 英文题名用短横线连接
- 原题号保持平台常用写法
- 题目名尽量短，不必完整保留副标题

## 五、三维难度记录规范

每道题从三个维度评分：

| 维度 | 记号 | 分制 | 核心问题 |
|---|---|---:|---|
| 知识点难度 | K | 1～5 | 学生需要学过什么？ |
| 思维难度 | T | 1～7 | 学过之后能不能想到？ |
| 实现难度 | I | 1～5 | 想到之后能不能写对？ |

统一记作：

```text
K / T / I
```

例如：

```text
2 / 4 / 2
3 / 5 / 3
3 / 1 / 3
```

核心解释：

```text
K：学没学过
T：想不想得到
I：写不写得稳
```

## 六、核心知识点与辅助思想

新增题目时必须区分：

```text
核心知识点
辅助思想
```

### 核心知识点

核心知识点用于决定知识点难度 K。

示例：

```text
差分
前缀和
枚举
线段树
最短路
树形 DP
网络流
```

### 辅助思想

辅助思想用于解释题目的思维难点，不参与知识点难度加分。

示例：

```text
区间贡献
乘积上界
反向处理
配对约束
状态压缩观察
贪心证明
分类讨论
```

重要规则：

```text
辅助思想不参与知识点难度加分。
```

例如：

```text
差分 + 区间贡献
```

不应直接打成知识点难度 4。更合理的是：

```text
核心知识点：差分
辅助思想：区间贡献，配对约束
知识点难度：2
思维难度：5
实现难度：3
三维难度：2 / 5 / 3
```

---

## 七、三维难度评分流程

### 第一步：确定核心知识点

只选最核心的一个或少数几个知识点。

示例：

```text
核心知识点：差分
辅助思想：区间贡献，配对约束
```

不要把辅助思想算进知识点难度。

---

### 第二步：判断知识点难度 K

问：

```text
学生是否需要上过一个完整专题课？
```

如果不需要，通常 K 不超过 2。

常见参考：

```text
K1：语法与基础操作
K2：基础算法套路
K3：标准提高组专题
K4：进阶专题或明显多专题组合
K5：省选 / NOI 高阶知识
```

---

### 第三步：判断思维难度 T

问：

```text
学生学过核心知识点后，能不能自然想到？
```

常见参考：

```text
T1：直接套用
T2：小变化
T3：常规关键性质
T4：关键优化 / 非直接性质
T5：明显模型转化
T6：强转化 / 多步性质
T7：非常规 / 省选级思维
```

---

### 第四步：判断实现难度 I

问：

```text
学生想到做法后，代码是否容易 WA？
```

常见参考：

```text
I1：很短很稳
I2：少量细节
I3：容易 WA
I4：多模块复杂
I5：极难调试
```

---

### 第五步：给综合等级

综合等级用于教学阶段定位，不直接等于三维难度最大值。

问：

```text
这题适合放在哪个阶段训练？
```

## 九、题目档案难度说明规范

每道 A 类或 B 类题都建议写具体说明，不要写套话。

推荐格式：

```markdown
## 三、难度评估

| 维度 | 分数 | 说明 |
|---|---:|---|
| 知识点难度 K | 2 | 核心知识点是差分，属于基础算法套路，不需要高级数据结构。 |
| 思维难度 T | 5 | 需要把每一对数的修改次数转化为区间贡献，这是本题主要难点。 |
| 实现难度 I | 3 | 需要处理贡献区间边界、数组下标和多组数据清空，容易出现 WA。 |
| 三维难度 | 2 / 5 / 3 | K / T / I。 |
| 综合等级 | L4～L5 | 适合作为差分专题中的提高训练题或讲评题。 |
```

## 十、新增题目流程

1. 在 `题目总表.md` 中登记基本信息，分配内部编号。
2. 根据题目分类（A/B/C）决定整理深度。
3. 确定主标签、核心知识点、辅助思想。
4. 按 `K / T / I` 规则评估三维难度。
5. 给出综合等级。
6. A类/B类题：在 `题目档案/` 下创建对应目录。
7. 按 `08-模板/题目档案模板.md` 填写各文件。
8. 更新总表中的链接和整理状态。
9. 如需加入题单，在 `题单/` 对应文件中添加。
10. 若后续讲评中发现难度偏差，应在维护日志中记录修正原因。

## 十一、难度校准优先规则

以下题型容易被打高，应特别注意：

```text
前缀和变形
差分区间贡献
枚举剪枝
简单贪心
数学边界题
输出方案题
```

校准原则：

```text
基础套路题的知识点难度默认不超过 2。
题目难在想法时，提高思维难度 T。
题目难在边界和代码时，提高实现难度 I。
不要因为题目难，就直接提高知识点难度 K。
```

## 十二、常见误判修正规则

### 误判 1：把思维难度算进知识点难度

错误：

```text
差分题很难 → K4
```

正确：

```text
差分是 K2，难想则提高 T。
```

---

### 误判 2：把实现细节算进知识点难度

错误：

```text
边界很多 → K4
```

正确：

```text
边界很多 → 提高 I，不提高 K。
```

---

### 误判 3：看到数学就打高

错误：

```text
有数学推导 → K4
```

正确：

```text
如果只是等差、等比、gcd、取模，通常 K2；
如果涉及矩阵、数论高级结论、组合计数高级工具，才考虑 K4 或 K5。
```

---

### 误判 4：看到 DP 就打高

错误：

```text
有 DP → K4
```

正确：

```text
线性 DP、背包 DP、树形 DP 基础通常 K3；
状压 DP、数位 DP 才通常 K4；
复杂 DP 优化才 K5。
```

---

### 误判 5：看到线段树就把思维打高

错误：

```text
线段树模板 → 3 / 4 / 3
```

正确：

```text
线段树模板 → 3 / 1 / 3
```

线段树模板是知识点和实现难，不是思维难。

## 十三、入库检查清单

每道题入库或校准后，检查下面 10 个问题：

```text
1. 知识点分是不是只由核心知识点决定？
2. 有没有把辅助思想误算进知识点？
3. 有没有把边界细节误算进知识点？
4. 如果 K ≥ 3，是否真的需要完整专题课？
5. 如果 T ≥ 5，是否真的存在明显模型转化？
6. 如果 T ≥ 6，是否真的需要多步性质或隐蔽建模？
7. 如果 I ≥ 4，代码是否真的有多模块复杂性？
8. 综合等级是否符合教学阶段？
9. 难度说明是否具体到本题，而不是套话？
10. 同类型题是否和已有题目分数一致？
```

## 十四、题面版权与使用说明

- 优先保留原题链接，题面文件中标注来源。
- 题面可整理为教学自用中文版本。
- 不建议大规模复制受版权保护的原文。
- 用于内部教学时应保留来源平台、题号、原题链接。
- 发送给 GPT 分析前注意脱敏。
