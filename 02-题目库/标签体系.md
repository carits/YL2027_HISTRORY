# 信息学题目标签体系设计标准

## 一、设计目标

本标准用于对信息学竞赛题目进行统一标签化管理，方便后续完成以下工作：

- 题目库分类管理
- 比赛与训练题单设计
- 学生做题情况统计
- 学生知识点掌握情况分析
- 错因统计与复盘
- GPT 辅助生成阶段学习报告
- 个性化训练计划推荐

本体系的核心思想是：

```text
一题一主类
多级知识点
三维难度评分
能力训练补充
常见错误沉淀
教学用途标记
```

其中最重要的三个维度是：

```text
知识点难度
思维难度
实现难度
```

这三个维度分别回答：

```text
这题需要学过什么？
学过之后能不能想到？
想到之后能不能写对？
```

---

## 二、题目标签总结构

每道题建议包含以下 6 类标签：

```text
1. 基础信息标签
2. 知识点标签
3. 难度三维标签
4. 能力训练标签
5. 常见错误标签
6. 教学使用标签
```

其中，题目库中最核心的字段为：

```text
平台
题号
题目名
题目链接
题解链接
主标签
子标签
知识点难度
思维难度
实现难度
能力标签
常见错误标签
教学标签
备注
```

---

## 三、三维难度评分总则

每个维度均采用 1～5 分。

不建议使用 10 分制，因为过细的分数会导致老师之间评分不稳定，也不利于长期维护。

| 分数 | 等级 | 含义 |
|---:|---|---|
| 1 | 入门 | 非常基础，适合刚接触相关内容的学生 |
| 2 | 基础 | 常规基础题，需要掌握常见套路 |
| 3 | 提高 | 需要系统学习专题知识，适合提高组训练 |
| 4 | 进阶 | 需要较强专题能力或多个知识点组合 |
| 5 | 高阶 | 省选/NOI 级别，知识、思维或实现要求很高 |

# 四、知识点难度标准

## 4.1 定义

知识点难度表示：

> 这道题所需要的算法、数据结构、数学工具本身有多高级。

它主要回答：

```text
学生有没有学过这个知识点？
这个知识点本身是不是高阶内容？
是否需要多个知识点组合？
```

## 4.2 评分标准

| 分数 | 等级 | 判断标准 |
|---:|---|---|
| 1 | 入门知识 | 只需要语法、模拟、简单枚举、排序、前缀和等基础内容 |
| 2 | 基础算法 | 需要常见基础算法或模板，如二分、贪心、DFS、BFS、并查集、简单 DP |
| 3 | 提高知识 | 需要较系统的专题知识，如线段树、树状数组、最短路、背包、树形 DP、拓扑排序 |
| 4 | 进阶知识 | 需要多个专题组合，或较复杂算法，如主席树、网络流、字符串高级算法、状压 DP、数位 DP |
| 5 | 高阶知识 | 需要省选/NOI 级知识，如动态树、点分治、斜率优化、分治优化、生成函数、复杂博弈、复杂图论 |

## 4.3 示例

| 题目类型 | 知识点难度 |
|---|---:|
| 简单模拟 | 1 |
| 排序后贪心 | 2 |
| 二分答案 + check | 2 |
| 单源最短路 Dijkstra | 3 |
| 线段树区间修改 | 3 |
| 树形 DP | 3 |
| 状压 DP | 4 |
| 主席树 | 4 |
| 网络流建图 | 4 |
| 点分治 | 5 |
| 斜率优化 DP | 5 |

# 五、思维难度标准

## 5.1 定义

思维难度表示：

> 从题面想到正确解法的难度。

它主要回答：

```text
这个题是不是容易看出模型？
是否需要转化？
是否需要观察性质？
是否需要证明贪心、单调性或等价关系？
```

注意：思维难度和知识点难度不是一回事。

有些题知识点很简单，但很难想到。例如：

```text
只用排序 + 贪心，但性质很隐蔽
知识点难度可能是 2
思维难度可能是 4
```

## 5.2 评分标准

| 分数 | 等级 | 判断标准 |
|---:|---|---|
| 1 | 直接套用 | 题面几乎直接提示做法，看完就知道用什么 |
| 2 | 轻微转化 | 需要简单分析，但模型比较明显 |
| 3 | 中等推导 | 需要发现关键性质，或从暴力优化到正解 |
| 4 | 较强思维 | 需要明显的转化、分类讨论、反向思考、构造或证明 |
| 5 | 高思维 | 解法非常隐蔽，需要强观察、复杂性质、非常规转化或多步推理 |

## 5.3 具体判断细则

### 1 分：直接题

特征：

```text
题目基本明示算法
学生学过模板后可以直接写
几乎不需要额外推导
```

例子：

```text
给区间加、区间求和 → 线段树模板
给无权图最短路 → BFS
给若干数求最大值 → 模拟
```

### 2 分：轻微变化题

特征：

```text
算法方向明显
需要一点点处理
有常见套路
```

例子：

```text
排序后贪心
二分答案后写 check
前缀和处理区间和
简单 DFS 枚举方案
```

### 3 分：需要分析题

特征：

```text
不能一眼看出完整做法
需要从暴力出发优化
需要发现一个关键性质
需要设计状态或维护信息
```

例子：

```text
区间 DP 状态设计
树形 DP 转移设计
Dijkstra 变形建图
线段树维护多个信息
```

### 4 分：明显思维题

特征：

```text
需要较强转化
需要证明单调性或贪心正确性
需要复杂分类讨论
需要把题意转成图、区间或状态
```

例子：

```text
贪心证明不明显
DP 状态压缩较难
图论建模隐蔽
反向处理 / 离线处理
整体二分思想
```

### 5 分：高难思维题

特征：

```text
解法非常不自然
关键性质很难观察
需要多步转化
常规套路不容易套上
即使知道知识点也不一定想得到
```

例子：

```text
复杂构造题
高阶计数题
复杂博弈题
需要多个性质串联的省选题
```

# 六、实现难度标准

## 6.1 定义

实现难度表示：

> 想到解法后，把代码写正确的难度。

它主要回答：

```text
代码量大不大？
细节多不多？
边界容易错吗？
调试难不难？
数据结构维护复杂吗？
```

注意：实现难度独立于知识点难度。

例如：

```text
线段树模板：
知识点难度 3
思维难度 1
实现难度 3

数学结论题：
知识点难度 4
思维难度 4
实现难度 1
```

## 6.2 评分标准

| 分数 | 等级 | 判断标准 |
|---:|---|---|
| 1 | 很容易写 | 代码短，边界少，基本不容易写挂 |
| 2 | 基础实现 | 有一定细节，但模板清楚，调试成本低 |
| 3 | 中等实现 | 代码结构较多，边界较多，容易出现常见 bug |
| 4 | 复杂实现 | 多模块、多数据结构、多状态，调试成本高 |
| 5 | 极难实现 | 代码量大，细节极多，稍有错误难以定位，比赛中很容易写挂 |

## 6.3 具体判断细则

### 1 分：简单实现

特征：

```text
几十行以内
单循环或简单判断
没有复杂边界
```

例子：

```text
简单模拟
简单数学公式
数组遍历统计
```

### 2 分：基础实现

特征：

```text
有一点逻辑分支
需要排序、二分、简单 DFS/BFS
代码量不大
```

例子：

```text
排序贪心
二分答案
简单图搜索
简单背包
```

### 3 分：中等实现

特征：

```text
需要维护数据结构
需要递归、多数组、多状态
边界容易错
```

例子：

```text
线段树
树状数组
最短路
树形 DP
区间 DP
```

### 4 分：复杂实现

特征：

```text
多个模块组合
代码量明显增大
边界和状态多
调试成本较高
```

例子：

```text
线段树 + 离散化 + 扫描线
图论建模 + 最短路
状压 DP + 预处理
网络流建图
字符串哈希 + 二分
```

### 5 分：极难实现

特征：

```text
代码长
细节密集
复杂数据结构
很难通过样例发现错误
比赛中高概率写挂
```

例子：

```text
动态树
可持久化复杂结构
复杂树链剖分
复杂几何
大型分类讨论构造
```

# 七、三维难度组合解释

每道题建议记录成：

```text
知识点难度 / 思维难度 / 实现难度
```

例如：

```text
3 / 1 / 3
```

表示：

```text
知识点是提高级
思维比较直接
实现中等偏复杂
```

## 7.1 常见组合解释

| 三维难度 | 题目特点 |
|---|---|
| 1 / 1 / 1 | 入门模拟题 |
| 2 / 2 / 2 | 基础训练题 |
| 3 / 1 / 3 | 模板题，例如线段树模板 |
| 2 / 4 / 2 | 知识点简单但思维巧妙的贪心/构造 |
| 4 / 2 / 4 | 高级模板或专题训练题 |
| 3 / 4 / 3 | 提高组难题，重点考转化 |
| 5 / 3 / 5 | 省选级复杂数据结构题 |
| 4 / 5 / 2 | 数学/结论/构造型思维题 |
| 2 / 3 / 5 | 知识点不难，但分类讨论极多、实现复杂 |

# 十二、综合等级设计

除了三维难度外，可以额外给每道题一个综合等级。

综合等级不应替代三维难度，而是用于题单排序和训练分层。

## 12.1 推荐综合等级

```text
L1 入门
L2 普及-
L3 普及+
L4 提高-
L5 提高
L6 提高+
L7 省选-
L8 省选
L9 省选+
L10 NOI
```

## 12.2 综合等级与三维难度的粗略关系

| 综合等级 | 三维难度大致范围 |
|---|---|
| L1 | 三维基本都是 1 |
| L2 | 主要是 1～2 |
| L3 | 有 2，少量 3 |
| L4 | 出现稳定的 3 |
| L5 | 多个维度达到 3 |
| L6 | 至少一个维度达到 4 |
| L7 | 多个维度达到 4 |
| L8 | 有 5，或者 4 / 4 / 4 |
| L9 | 多个维度达到 5 |
| L10 | 顶级综合题 |

# 十五、评分快速判断规则

为了避免不同老师打分不一致，可以使用以下快速判断规则。

## 15.1 知识点难度怎么判断

```text
看这题需要学生学过什么。
```

如果学生不会这个专题，就基本做不了，说明知识点难度高。

---

## 15.2 思维难度怎么判断

```text
看学生知道相关知识点后，能不能想到怎么用。
```

如果告诉他“这是线段树题”后就会做，思维难度低。

如果告诉他“这是贪心题”后还是想不到贪心策略，思维难度高。

---

## 15.3 实现难度怎么判断

```text
看学生已经知道完整做法后，代码会不会写挂。
```

如果思路清楚但很容易 WA、TLE、RE，说明实现难度高。

# 十七、与学生画像的关联方式

这套标签体系的价值在于可以反向分析学生问题。

例如：

## 情况 1：学生经常错在 2 / 4 / 2 的题

说明：

```text
知识点不难，代码也不难，但思维转化弱。
```

可能问题：

```text
性质观察能力不足
贪心证明能力不足
不会从暴力推导正解
缺少反例构造意识
```

训练建议：

```text
多安排思维题、贪心题、构造题
做题时要求先写出错误思路和反例
课堂上加强“为什么这样做”的证明训练
```

---

## 情况 2：学生经常错在 3 / 1 / 4 的题

说明：

```text
能看出做法，但代码写不稳定。
```

可能问题：

```text
模板不熟
代码结构混乱
边界意识弱
调试能力不足
```

训练建议：

```text
减少新知识输入
加强模板默写和二刷
建立提交前检查清单
记录常见实现错误
```

---

## 情况 3：学生经常错在 4 / 2 / 4 的题

说明：

```text
进阶知识点掌握不扎实，且实现压力较大。
```

可能问题：

```text
专题学习不系统
只看过题解但没有形成稳定模板
复杂代码缺少分模块能力
```

训练建议：

```text
回到专题基础题
先做模板题和巩固题
再做综合题
避免直接上压轴题
```

# 十九、最终总结

本题目标签体系的核心不是单纯给题目分类，而是服务于教学分析。

最终应形成如下闭环：

```text
题目打标签
↓
比赛/训练引用题目
↓
学生产生做题记录和比赛记录
↓
系统按标签统计学生薄弱点
↓
老师补充重点评价
↓
GPT 生成阶段报告和训练建议
```

最核心的三句话：

```text
知识点难度：这题需要学过什么？
思维难度：学过之后能不能想到？
实现难度：想到之后能不能写对？
```

只要这三个维度稳定记录下来，后续对学生的分析就会比单纯记录“做了哪些题、考了多少分”精确很多。
