# 信息学题目标签体系设计标准

## 一、设计目标

本标准用于对信息学竞赛题目进行统一标签化管理，方便后续完成以下工作：

- 题目库分类管理
- 比赛与训练题单设计
- 学生做题情况统计
- 学生知识点掌握情况分析
- 错因统计与复盘
- GPT 辅助生成阶段学习报告
- 个性化训练计划推荐
- 长期沉淀教学素材

本体系的核心思想是：

```text
一题一主类
核心知识点明确
辅助思想单独记录
三维难度评分
能力训练补充
常见错误沉淀
教学用途标记
```

其中最重要的三个维度是：

```text
知识点难度 K
思维难度 T
实现难度 I
```

三个维度分别回答：

```text
K：这题需要学过什么？
T：学过之后能不能想到？
I：想到之后能不能写对？
```

---

## 二、题目标签总结构

每道题建议包含以下 6 类标签：

```text
1. 基础信息标签
2. 知识点标签
3. 三维难度标签
4. 能力训练标签
5. 常见错误标签
6. 教学使用标签
```

题目库中建议保留以下核心字段：

```text
平台
题号
题目名
题目链接
题解链接

主标签
核心知识点
辅助思想
子标签
细分标签

知识点难度 K
思维难度 T
实现难度 I
三维难度 K / T / I
综合等级

能力标签
常见错误标签
教学标签

训练重点
前置知识
适合阶段
备注
```

# 三、三维难度评分总则

## 3.1 分制

每道题从三个维度评分：

| 维度 | 记号 | 分制 | 核心问题 |
|---|---|---:|---|
| 知识点难度 | K | 1～5 | 学生需要学过什么？ |
| 思维难度 | T | 1～7 | 学过之后能不能想到？ |
| 实现难度 | I | 1～5 | 想到之后能不能写对？ |

统一记作：

```text
K / T / I
```

例如：

```text
2 / 4 / 2
3 / 5 / 3
3 / 1 / 3
```

---

## 3.2 最重要的防打高规则

### 规则 1：默认低分，满足门槛才升分

评分时不要凭“感觉难”直接打高。

每道题先默认：

```text
知识点难度：1
思维难度：1
实现难度：1
```

只有满足明确条件，才升到对应分数。

---

### 规则 2：知识点难度只看“前置知识门槛”

知识点难度不看题目难不难，只看：

```text
如果学生没有学过这个知识点，是否基本无法开始？
```

如果只是题目里“用到一点思想”，不应该提高知识点难度。

下面这些一般不单独提高知识点难度：

```text
乘积上界
区间贡献
反向处理
分类讨论
贡献统计
贪心证明
单调性观察
枚举范围压缩
式子变形
边界分析
```

这些应该放到：

```text
思维难度
辅助思想标签
能力标签
```

---

### 规则 3：题目难，不等于知识点难

错误打分：

```text
这题差分很难，所以知识点难度 4。
```

正确打分：

```text
差分本身是基础算法，知识点难度 2；
难在区间贡献转化，所以思维难度 5。
```

---

### 规则 4：没有正式专题门槛，不给知识点 3

打知识点难度 3 之前必须问：

```text
这题是否需要学生上过一个完整专题课？
```

如果答案是否，那么知识点难度通常不超过 2。

---

### 规则 5：综合等级不等于最高分

例如：

```text
2 / 5 / 2
```

不代表一定是高等级题。

它可能只是一个基础知识点下的思维题。

综合等级要看整体教学阶段，而不是直接取三维难度的最大值。

---

## 3.3 推荐字段设计

建议每道题保存以下难度字段：

```text
核心知识点
辅助思想
知识点难度
思维难度
实现难度
三维难度
综合等级
难度说明
```

推荐格式：

```text
核心知识点：差分
辅助思想：区间贡献，配对约束，修改次数统计
知识点难度：2
思维难度：5
实现难度：3
三维难度：2 / 5 / 3
综合等级：L5
```

# 四、知识点难度 K：1～5

知识点难度表示：

```text
这道题所需的最高前置算法或数据结构门槛。
```

它只看“学没学过”，不看“想不想得到”。

---

## K1：语法与基础操作

### 定义

只需要基础语法和简单操作，不需要专门算法专题。

### 满足条件

```text
会输入输出
会 if / for / while
会数组遍历
会简单字符串处理
会简单排序
会简单数学公式
```

### 常见内容

```text
简单模拟
数组统计
简单排序
简单字符串判断
简单数学公式
简单条件判断
```

### 典型题型

```text
统计最大值 / 最小值
闰年判断
简单字符串包含
简单排序输出
简单坐标移动
```

### 判断标准

```text
不需要老师专门讲算法专题，只要会语法就能做。
```

---

## K2：基础算法套路

### 定义

需要学过一个基础算法套路，但不是提高组专题。

### 常见内容

```text
前缀和
差分
枚举
剪枝枚举
双指针
尺取法基础
简单贪心
简单二分
简单构造
简单 DFS / BFS
map / set / priority_queue 基础用法
gcd
快速幂
简单排列组合
```

### 典型题型

```text
前缀和求区间统计
差分做区间加
排序后贪心
堆模拟多个窗口
枚举时利用简单上界剪枝
map 统计出现次数
简单 BFS 搜索
```

### 判断标准

```text
这题需要知道一个基础套路，但不需要完整提高组专题课。
```

### 封顶规则

下面这些题，除非明显结合复杂模型，否则知识点难度默认不超过 2：

```text
前缀和
差分
枚举剪枝
简单贪心
简单二分
简单堆
简单 map / set
简单 gcd / 快速幂
```

---

## K3：标准提高组专题

### 定义

需要系统学习过一个正式专题，没学过基本做不了。

### 常见内容

```text
线段树
树状数组
并查集
二维前缀和
ST 表
单源最短路
拓扑排序
LCA
背包 DP
线性 DP
区间 DP 基础
树形 DP 基础
字符串哈希
KMP
```

### 典型题型

```text
线段树维护区间信息
二维前缀和处理矩形查询
Dijkstra 最短路
并查集维护连通性
背包状态转移
树形 DP
LCA 查询
KMP 匹配
```

### 判断标准

```text
这题是否需要专门讲一节专题课？
```

如果需要，通常是 K3。

---

## K4：进阶专题或明显多专题组合

### 定义

需要较强专题能力，或者两个以上专题深度组合。

### 常见内容

```text
树链剖分
主席树
可持久化数据结构
状压 DP
数位 DP
网络流
最小割
费用流
强连通分量复杂应用
AC 自动机
后缀数组
复杂线段树维护
扫描线 + 线段树
DP 优化基础
```

### 典型题型

```text
树链剖分维护路径信息
主席树查询区间第 k 小
状压 DP 处理集合状态
网络流建图
扫描线加线段树求面积并
AC 自动机上 DP
```

### 判断标准

```text
这题是否可以作为提高+ 或省选- 专题训练？
```

如果是，通常是 K4。

---

## K5：省选 / NOI 高阶知识

### 定义

需要高阶专题，普通提高组主线训练一般不会默认覆盖。

### 常见内容

```text
点分治
动态树
斜率优化
决策单调性优化
生成函数
复杂组合计数
复杂博弈
高阶字符串结构
复杂计算几何
大型综合数据结构
复杂网络流优化
```

### 典型题型

```text
点分治统计路径
LCT 动态维护森林
斜率优化 DP
生成函数计数
复杂博弈结论
复杂几何分类
```

### 判断标准

```text
没学过省选专题，基本无法独立推出完整算法。
```

# 五、思维难度 T：1～7

思维难度表示：

```text
学生已经学过相关知识点后，从题面想到正确做法的难度。
```

思维难度不看算法本身高级不高级，而看：

```text
是否需要观察性质
是否需要模型转化
是否需要证明
是否需要从暴力优化到正解
是否需要非常规思考
```

---

## T1：直接套用

### 定义

题面几乎直接提示做法，学过对应知识点后可以直接写。

### 特征

```text
题意和算法模型几乎一致
不需要额外性质
不需要明显转化
```

### 典型题型

```text
区间求和 → 前缀和
区间加 → 差分
无权图最短路 → BFS
线段树模板
最短路模板
背包模板
```

### 示例

```text
线段树模板：3 / 1 / 3
简单前缀和：2 / 1 / 1
```

---

## T2：小变化

### 定义

方向明显，只需要一点处理。

### 特征

```text
课堂例题的轻微变形
算法方向容易看出
只有一个很小的处理点
```

### 典型题型

```text
排序后简单贪心
堆模拟多窗口
二分答案写 check
枚举候选后判断
简单前缀和换统计方式
```

### 示例

```text
堆模拟接水：2 / 2 / 2
简单排序贪心：2 / 2 / 1
```

---

## T3：常规关键性质

### 定义

需要发现一个常规性质，但这个性质属于训练中经常见到的套路。

### 特征

```text
需要式子改写
需要前缀状态统计
需要枚举一边确定另一边
需要根据最大值 / 最小值分类
需要用 map 记录历史状态
```

### 典型题型

```text
子数组和转前缀差值
哈希统计前缀状态
简单分类讨论
固定一个变量推出另一个变量
```

### 示例

```text
前缀和变形：2 / 3 / 2
哈希统计前缀状态：2 或 3 / 3 / 2
```

---

## T4：关键优化 / 非直接性质

### 定义

不是简单套套路，需要从暴力出发优化，或者利用一个不太直观的限制。

### 特征

```text
暴力显然，但需要剪枝到可过
需要利用值域或下标上界
需要压缩枚举范围
需要看出只需考虑有限状态
```

### 典型题型

```text
乘积上界剪枝
枚举范围压缩
利用 i + j 的上界
只枚举可能有效的候选
```

### 示例

```text
CF1541B Pleasant Pairs：2 / 4 / 2
```

说明：

```text
知识点只是枚举；
难点是发现 a_i * a_j = i + j 且 i + j 不超过 2n，从而剪枝。
```

---

## T5：明显模型转化

### 定义

题意需要整体换一种看法，不是局部优化，而是转成另一种贡献、状态或判定。

### 特征

```text
区间贡献
反向处理
贡献统计
0/1/2 次修改范围
贪心证明较明显
把原问题转成判定问题
把操作过程转成统计问题
```

### 典型题型

```text
差分区间贡献
反向维护答案
贡献法统计
较强贪心证明
图论隐式建模基础版
```

### 示例

```text
CF1343D Constant Palindrome Sum：2 / 5 / 3
```

说明：

```text
知识点是差分；
难点是把每一对数转成 0/1/2 次修改的区间贡献。
```

---

## T6：强转化 / 多步性质

### 定义

需要多个性质连续推出，或者核心转化比较隐蔽。

### 特征

```text
需要多步推理
需要较难证明
需要隐式建模
需要设计不明显的 DP 状态
需要把多个性质串起来
```

### 典型题型

```text
复杂贪心证明
图论隐式建模
较难 DP 状态设计
构造题核心规律不明显
组合计数拆贡献
```

### 适合定位

```text
提高组难题
比赛讲评题
赛后复盘题
专题压轴题
```

---

## T7：非常规 / 省选级思维

### 定义

关键性质非常隐蔽，需要多步转化，常规套路很难套上。

### 特征

```text
看完题解会觉得很难想到
核心结论不自然
需要非常规观察
需要多层转化
需要省选级思维训练
```

### 典型题型

```text
高难构造
复杂计数
复杂博弈
省选级性质题
非套路转化题
多结论串联题
```

# 六、实现难度 I：1～5

实现难度表示：

```text
想到做法后，把代码写正确的难度。
```

它主要看：

```text
代码结构是否复杂
边界是否多
下标是否容易错
是否有多组数据清空
是否需要复杂数据结构维护
是否难以调试
```

---

## I1：很短很稳

### 定义

代码短，边界少，几乎不容易写挂。

### 特征

```text
一个循环或简单判断
没有复杂数组
没有复杂边界
不需要特殊数据类型处理
```

### 典型题型

```text
简单模拟
简单数学公式
数组遍历统计
简单字符串判断
```

---

## I2：少量细节

### 定义

有一点实现细节，但代码结构清楚，调试成本低。

### 特征

```text
需要排序
需要 map / set / heap
需要简单函数
有少量边界
代码量不大
```

### 典型题型

```text
简单贪心
堆模拟
简单二分
简单 DFS / BFS
简单枚举剪枝
```

---

## I3：容易 WA

### 定义

代码不一定很长，但下标、边界、状态较多，容易写错。

### 特征

```text
多数组
多状态
多组数据清空
需要 long long
需要处理特殊情况
区间边界容易错
状态转移顺序容易错
```

### 典型题型

```text
二维前缀和
差分区间贡献
输出方案
线段树基础模板
树形 DP 基础题
字符串操作细节题
数学边界题
```

---

## I4：多模块复杂

### 定义

代码结构明显复杂，有多个模块组合，调试成本较高。

### 特征

```text
预处理 + 主算法 + 查询
多个数据结构配合
复杂状态维护
边界很多
代码量明显增加
```

### 典型题型

```text
扫描线 + 线段树
状压 DP + 预处理
图建模 + 最短路
网络流建图
复杂线段树维护多个信息
树链剖分
```

---

## I5：极难调试

### 定义

代码长、细节密集，一个小错误很难通过样例发现。

### 特征

```text
大型数据结构
复杂分类讨论
复杂几何
复杂树上维护
可持久化复杂结构
动态维护
```

### 典型题型

```text
动态树
复杂树剖
复杂可持久化数据结构
复杂计算几何
大型分类讨论构造
高阶字符串结构
```

# 七、知识点标签体系

知识点标签建议分三级：

```text
一级标签：大方向
核心知识点：决定知识点难度 K 的主要算法或工具
辅助思想：不参与 K 加分，只用于解释思维难点
```

---

## 7.1 一级标签

建议固定为以下类别：

```text
基础语法
模拟
枚举
贪心
二分
递归与搜索
动态规划
数据结构
图论
树
字符串
数学
组合计数
计算几何
博弈论
构造
交互
随机化
```

---

## 7.2 核心知识点与辅助思想示例

### 基础算法类

```text
核心知识点：前缀和
辅助思想：前缀差值，状态计数，区间统计

核心知识点：差分
辅助思想：区间贡献，配对约束，修改次数统计

核心知识点：枚举
辅助思想：乘积上界，剪枝枚举，固定一维推另一维

核心知识点：贪心
辅助思想：交换论证，反悔，局部最优证明
```

### 数据结构类

```text
核心知识点：线段树
辅助思想：懒标记，区间合并，复杂信息维护

核心知识点：树状数组
辅助思想：差分维护，前缀贡献，离线查询

核心知识点：并查集
辅助思想：连通性维护，反向加边，种类并查集
```

### 图论类

```text
核心知识点：最短路
辅助思想：状态扩展，隐式建图，分层图

核心知识点：最小生成树
辅助思想：边权排序，瓶颈性质，Kruskal 重构树

核心知识点：网络流
辅助思想：容量建模，最小割转化，二分图匹配转化
```

### 动态规划类

```text
核心知识点：线性 DP
辅助思想：状态设计，转移优化，前缀状态

核心知识点：背包 DP
辅助思想：容量约束，方案计数，滚动数组

核心知识点：树形 DP
辅助思想：子树合并，换根，父子状态转移
```

### 数学类


# 十一、综合等级 L1～L10

综合等级用于题单排序和学生阶段定位，不替代三维难度。

推荐等级：

```text
L1 入门
L2 普及-
L3 普及+
L4 提高-
L5 提高
L6 提高+
L7 省选-
L8 省选
L9 省选+
L10 NOI
```

---

## 11.1 综合等级判断规则

### L1 入门

```text
三维大致：1 / 1～2 / 1
```

适合刚入门。

---

### L2 普及-

```text
三维大致：1～2 / 2～3 / 1～2
```

有基础套路，但不复杂。

---

### L3 普及+

```text
三维大致：2 / 3～4 / 2
```

有一定思维点，适合普及组选手训练。

---

### L4 提高-

```text
三维大致：3 / 3～4 / 3
或 2 / 5 / 3
```

开始出现提高组专题，或者基础知识点下的较强思维题。

---

### L5 提高

```text
三维大致：3 / 5 / 3
或 3 / 4 / 4
```

适合提高组核心训练。

---

### L6 提高+

```text
三维大致：3～4 / 5～6 / 3～4
```

需要较强专题能力或明显思维压力。

---

### L7 省选-

```text
三维大致：4 / 5～6 / 4
或 3 / 6～7 / 4
```

接近省选训练题。

---

### L8 省选

```text
三维大致：4～5 / 6 / 4～5
```

省选级综合题。

---

### L9 省选+

```text
三维大致：5 / 6～7 / 5
```

高阶综合题。

---

### L10 NOI

顶级综合题，知识、思维、实现至少两个维度达到最高档。

# 十三、评分流程

每道题按下面顺序评分。

## 第一步：确定核心知识点

只选最核心的一个或少数几个知识点。

示例：

```text
核心知识点：差分
辅助思想：区间贡献，配对约束
```

不要把辅助思想算进知识点难度。

---

## 第二步：判断知识点难度 K

问：

```text
学生是否需要上过一个完整专题课？
```

如果不需要，通常 K 不超过 2。

---

## 第三步：判断思维难度 T

问：

```text
学生学过核心知识点后，能不能自然想到？
```

如果只是小变化，T2。

如果要发现一个关键性质，T3～T4。

如果要明显转化，T5。

如果要多步性质，T6～T7。

---

## 第四步：判断实现难度 I

问：

```text
学生想到做法后，代码是否容易 WA？
```

如果边界少，I1～I2。

如果多数组、多状态、下标复杂，I3。

如果多模块组合，I4。

如果非常难调试，I5。

---

## 第五步：给综合等级

综合等级用于教学阶段，不直接等于三维最大值。

问：

```text
这题适合放在哪个阶段训练？
```

---

# 十四、典型例子校准

## 例 1：堆模拟接水

```text
核心知识点：priority_queue
辅助思想：多窗口模拟
知识点难度：2
思维难度：2
实现难度：2
三维难度：2 / 2 / 2
综合等级：L2
```

解释：

```text
需要会堆，但不需要高级算法；
思维只是把每个水龙头的结束时间放入小根堆；
实现细节不多。
```

---

## 例 2：Pleasant Pairs

```text
核心知识点：枚举
辅助思想：乘积上界，剪枝枚举
知识点难度：2
思维难度：4
实现难度：2
三维难度：2 / 4 / 2
综合等级：L3
```

解释：

```text
知识点只是枚举；
难点是由 i + j 的上界推出乘积上界；
实现不复杂。
```

---

## 例 3：Constant Palindrome Sum

```text
核心知识点：差分
辅助思想：区间贡献，配对约束，修改次数统计
知识点难度：2
思维难度：5
实现难度：3
三维难度：2 / 5 / 3
综合等级：L4～L5
```

解释：

```text
差分本身是基础算法；
核心难点是把每一对数转成 0/1/2 次修改的区间贡献；
实现上有不少边界。
```

---

## 例 4：二维前缀和统计骨牌

```text
核心知识点：二维前缀和
辅助思想：横向骨牌统计，纵向骨牌统计，矩形边界
知识点难度：3
思维难度：4
实现难度：3
三维难度：3 / 4 / 3
综合等级：L4
```

解释：

```text
二维前缀和算标准提高组专题；
难点在于不是统计点，而是统计相邻点对；
实现边界容易错。
```

---

## 例 5：线段树模板

```text
核心知识点：线段树
辅助思想：无
知识点难度：3
思维难度：1
实现难度：3
三维难度：3 / 1 / 3
综合等级：L4
```

解释：

```text
知识点门槛高；
但题面基本直接对应模板；
实现需要写线段树，容易出错。
```

---

## 例 6：图论建模最短路

```text
核心知识点：最短路
辅助思想：隐式建图，状态扩展
知识点难度：3
思维难度：5
实现难度：3
三维难度：3 / 5 / 3
综合等级：L5
```

解释：

```text
最短路本身是 K3；
难点在于把题意转成图；
实现通常不至于特别复杂，但容易建错边。
```

# 十五、三维难度说明模板

每道题的难度说明建议不要写套话，按下面格式写。

```markdown
## 难度评估

| 维度 | 分数 | 说明 |
|---|---:|---|
| 知识点难度 K | 2 | 核心知识点是差分，属于基础算法套路，不需要高级数据结构。 |
| 思维难度 T | 5 | 需要把每一对数的修改次数转化为区间贡献，这是本题主要难点。 |
| 实现难度 I | 3 | 需要处理贡献区间边界、数组下标和多组数据清空，容易出现 WA。 |
| 三维难度 | 2 / 5 / 3 | K / T / I。 |
| 综合等级 | L4～L5 | 适合作为差分专题中的提高训练题或讲评题。 |
```

# 十六、常见误判修正规则

## 误判 1：把思维难度算进知识点难度

错误：

```text
差分题很难 → K4
```

正确：

```text
差分是 K2，难想则提高 T。
```

---

## 误判 2：把实现细节算进知识点难度

错误：

```text
边界很多 → K4
```

正确：

```text
边界很多 → 提高 I，不提高 K。
```

---

## 误判 3：看到数学就打高

错误：

```text
有数学推导 → K4
```

正确：

```text
如果只是等差、等比、gcd、取模，通常 K2；
如果涉及矩阵、数论高级结论、组合计数高级工具，才考虑 K4 或 K5。
```

---

## 误判 4：看到 DP 就打高

错误：

```text
有 DP → K4
```

正确：

```text
线性 DP、背包 DP、树形 DP 基础通常 K3；
状压 DP、数位 DP 才通常 K4；
复杂 DP 优化才 K5。
```

---

## 误判 5：看到线段树就把思维打高

错误：

```text
线段树模板 → 3 / 4 / 3
```

正确：

```text
线段树模板 → 3 / 1 / 3
```

线段树模板是知识点和实现难，不是思维难。

# 十七、入库时的最终检查清单

每道题打分后，检查下面 10 个问题：

```text
1. 知识点分是不是只由核心知识点决定？
2. 有没有把辅助思想误算进知识点？
3. 有没有把边界细节误算进知识点？
4. 如果 K ≥ 3，是否真的需要完整专题课？
5. 如果 T ≥ 5，是否真的存在明显模型转化？
6. 如果 T ≥ 6，是否真的需要多步性质或隐蔽建模？
7. 如果 I ≥ 4，代码是否真的有多模块复杂性？
8. 综合等级是否符合教学阶段？
9. 难度说明是否具体到本题，而不是套话？
10. 同类型题是否和已有题目分数一致？
```
